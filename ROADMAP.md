# VideoTranscript — Roadmap

## Сделано
- Загрузка локальных видео-файлов
- Выбор сегментов на таймлайне (IN/OUT маркеры)
- Оценка стоимости DeepGram до транскрипции
- Транскрипция через DeepGram Nova-3 (RU/EN/multi)
- Восстановление оригинальных таймкодов после вырезки сегментов
- Просмотр транскрипта с кликабельными таймкодами
- Структурирование через LLM: Summary / Thesis / Notes / Actions
- Мульти-провайдер LLM: Groq, Gemini, DeepSeek, Claude, Ollama
- Экспорт в Markdown
- SQLite для хранения истории
- Страница Settings с выбором провайдера и вводом ключей

## Ближайшее (UX)

### Форк уже готового решения
Посмотреть https://github.com/topraaman/tldr-video - установить, проверить как работает. Описать 2 пути: 1) как работает это готовое решение 2) как работает наше решение. Выписать плюсы и минусы каждого. Решить что можно выкинуть, а что оставить и интегрировать к себе 

### Транскрипт — группировка строк
- Сейчас слишком мелкая нарезка (~8 сек на блок)
- Группировать по спикеру, разрезать только при смене или превышении порога
- Слайдер: максимальный размер блока от 30 сек до 5 мин
- Диалог должен оставаться диалогом (Спикер 1 / Спикер 2 / Спикер 1...)

### Видимый лоадер при структурировании
- Сейчас кнопка просто пропадает — непонятно что происходит
- Добавить спиннер + текст "Анализирую..." + примерное время ожидания

### Автоскролл транскрипта
- При воспроизведении видео — подсвечивать и скроллить к текущей фразе
- Кликабельные таймкоды в Notes уже есть, нужна обратная связь

## Средний приоритет (функциональность)

### YouTube / URL импорт
- Поле для вставки URL (YouTube, Loom, Vimeo) рядом с дропзоной
- Скачивание через yt-dlp, дальше тот же pipeline
- Автоматическое определение языка из метаданных

### Удаление промо-сегментов
- Попросить LLM определить sponsor/outro/intro сегменты
- Предложить их исключить перед транскрипцией

### Повторное структурирование
- Возможность запустить Structure повторно с другим промптом
- Кастомный промпт ("сделай акцент на технических деталях", "выдели вопросы")
- История версий структурирования

### Редактирование транскрипта
- Inline-редактирование ошибок в словах (имена, термины)
- Сохранение правок в БД

### Экспорт форматов
- DOCX (как у tldr-video) — красиво, можно поделиться
- PDF
- Notion-совместимый формат (вставка блоками)

## Дистрибуция

### Desktop приложение (Electron / Tauri)
- Упаковать в .dmg (Mac) и .exe (Windows)
- ffmpeg внутри пакета — не нужно устанавливать отдельно
- API ключи через UI Settings (не через .env.local)
- Хранение ключей в системном keychain

### Для нетехнического пользователя
- Дефолт на Groq (бесплатный, регистрация по email)
- Онбординг: пошаговый экран при первом запуске
  1. Вставь Groq API key (ссылка на console.groq.com)
  2. Вставь DeepGram key (ссылка, есть бесплатный тариф)
  3. Загрузи первое видео
- Никакого терминала, никакого .env

## Низкий приоритет / Идеи

### Поиск по транскриптам
- Полнотекстовый поиск по всей библиотеке (SQLite FTS5)
- "Найди все видео где упоминается Quick Sort"

### Теги и категории
- Ручная и автоматическая (через LLM) разметка видео
- Фильтрация библиотеки по темам

### Шаблоны промптов
- Лекция / Встреча / Интервью / Подкаст
- Разные шаблоны дают разные структуры вывода

### Sharing
- Экспорт результата как публичная read-only ссылка
- Требует хостинга, не для локальной версии

### Локальная транскрипция (Whisper)
- Whisper через mlx-whisper (Apple Silicon) или faster-whisper
- Полностью офлайн, без DeepGram
- Медленнее, хуже русский язык, но бесплатно

### Мобильное приложение
- Нереалистично как нативное (ffmpeg, большие файлы)
- Возможно как PWA для просмотра уже готовых транскриптов
